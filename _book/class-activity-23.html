Answer:

db_recipe <- recipe(diabetes ~  glucose + insulin, data = db_train) %>%
  step_scale(all_predictors()) %>%
  step_center(all_predictors()) %>% 
  prep()
  1. Create your model specification and use tune() as a placeholder for the number of neighbors
Click for answer

Answer:

knn_spec <- nearest_neighbor(weight_func = "rectangular", 
                             engine = "kknn",
                             mode = "classification",
                             neighbors = tune())
  1. Split the db_train data set into v = 10 folds, stratified by diabetes
Click for answer

Answer:

db_vfold <- vfold_cv(db_train, v = 10, strata = diabetes)
  1. Create a grid of K values, the number of neighbors and run 10-fold CV on the k_vals grid, storing four performance metrics. The vizualization code is provided for your reference.
Click for answer

Answer:

k_vals <- tibble(neighbors = seq(from = 1, to = 40, by = 1))
knn_fit <- workflow() %>%
  add_recipe(db_recipe) %>%
  add_model(knn_spec) %>%
  tune_grid(
    resamples = db_vfold, 
    grid = k_vals,
    metrics = metric_set(yardstick::ppv, yardstick::accuracy, sens, spec),
    control = control_resamples(save_pred = TRUE))
cv_metrics <- collect_metrics(knn_fit) 
cv_metrics %>% group_by(.metric) %>% slice_max(mean)

29.2 Group Activity 2

  1. Let’s fit the logistic regression model.
set.seed(12345)
db_single <- db %>% select(diabetes, glucose)
db_split <- initial_split(db_single, prop = 0.80)

# Create training data
db_train <- db_split %>% training()

# Create testing data
db_test <- db_split %>% testing()

fitted_logistic_model <- logistic_reg() %>% # Call the model function
        # Set the engine/family of the model
        set_engine("glm") %>%
        # Set the mode
        set_mode("classification") %>%
        # Fit the model
        fit(diabetes~., data = db_train)

tidy(fitted_logistic_model)

  1. We are interested in predicting the diabetes status of patients depending on the amount of glucose. Verify that the glucose value of 143.11 gives the probability of having diabetes as 1/2.
Click for answer

Answer:

\[log\left(\frac{p}{1-p}\right) = \beta_0 + \beta_1x\]

(p <- round(exp(-5.61 + 0.0392* 143.11) / (1 + exp(-5.61 + 0.0392* 143.11)),2))

  1. What value of glucose is needed to have a probability of diabetes of 0.75?

Answer:

p <- 0.75
(x <- (log(p/(1-p)) - (-5.61))/0.0392)

  1. Make a classifier that classifies the diabetes status of new patients with a threshold of 0.75, i.e, a new patient is classified as negative if the estimated class probability is less than 0.75. Also, create a confusion matrix of the resulting predictions.

# Prediction Probabilities
